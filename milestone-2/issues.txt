We tried running the spark stream aggregator file and then collecting the data from drone by running kafka producer file.
We were able to collect the data accurately in kafka producer, but spark file was unable to perform calculations as fast as the data was flowing in.
So think of this like we collect first data set of 5 data points, spark begins its averages and other calculations on the first data set.
But, oh we received second data set already, so spark now switches to doing the calculations on second data set abandoning the first data set.
This keeps on going till the 120th data set which is the last one it collects. Since kafka producer does not collect any more data points.
Spark has enough time to run calculations on the last data set. So we see 119 empty files generated by spark file.
But we see an accurate calculated file by spark on the last data set.
This is due to threading issue, moreover spark is not able to perform calculations as fast as the data coming in.
This cause spark to keep jumping to new data point until it has enough time to do the calculation.
So how we address this issue is by collecting the data in one batch of 120 data points.
Then telling spark to perform calculations on those 120 data points after it has been collected.
Once we have all the analysis done with spark, we pass that to visualize aggregates file to generate plots for it.
This solution avoid all the previous threading issues and bypasses the complexity of making the spark and kafka run simultaneously live time.
